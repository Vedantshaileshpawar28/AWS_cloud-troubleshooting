# AWS Cloud Troubleshooting Labs

This repository contains hands-on AWS cloud troubleshooting labs designed to simulate real-world production incidents.

The goal of this project is to demonstrate practical skills required for Cloud Support Engineer and Junior DevOps roles, including:

- Linux system troubleshooting
- AWS EC2 networking and security
- Incident investigation and root cause analysis
- Clear technical documentation

## Tech Stack
- AWS EC2 (Ubuntu / Amazon Linux)
- Linux CLI
- Nginx
- Bash
- Git & GitHub

## Incident Labs

### Incident 01: High CPU Usage on EC2
Investigated and resolved excessive CPU utilization caused by a misbehaving process.

### Incident 02: Application Inaccessible Due to Security Group Misconfiguration
Diagnosed networking and security group rules preventing access to a web application.

### Incident 03: Disk Full Issue Caused by Log Files
Identified disk space exhaustion caused by uncontrolled log growth and applied remediation.

Each incident includes:
- Problem description
- Investigation steps
- Root cause analysis
- Resolution and prevention steps
